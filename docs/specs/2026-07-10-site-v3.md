# Spec — Site marketing v3 (roadmapped.work)

Statut : **à valider par Rémi**. Auteur : agent. Date : 2026-07-10.

## Contexte

La v2 du site est en ligne (hero XXL + oiseau sur le R, DA Vercel + fond pixel-art, copy revue, toggle thème, oiseau theme-aware). Retours de Rémi après mise en ligne : la première partie doit respirer (full-bleed) mais garder les grilles ; la démo embarquée est une vieille build de l'app et ne suit pas le thème sombre ; le hero peut rester gros mais moins bold ; « How it works » n'explique rien (juste 4 étapes en texte) ; le fond pixel-art ne plaît pas ; la copy ne dit pas LE problème qu'on résout ; on peut pousser la DA (bleu, avec légèreté). Ton de marque : voix de Rémi (`docs/tone-of-voice.md`) — direct, pince-sans-rire, anti-marketing méta, chute sèche, zéro superlatif.

Repos concernés : **site** `5e1y/roadmapped-site` (statique : `index.html`, `style.css`, `roadmapped-bird.js`, `demo/`) et **app** `roadmapped` (build de la démo via `npm run build:demo` → `dist-demo/`, recopiée dans `site/demo/` ; `index.demo.html` = entrée de la démo). Travail sur branche `site-v3`, merge → `main` déclenche le déploiement Cloudflare.

## Décisions

### D1 — Hero : la valeur = l'interface qui donne la vue d'ensemble
**Recadrage produit (validé Rémi)** : la valeur n'est PAS « des fichiers YAML » (ça c'est le mécanisme). La valeur = ton agent IA génère les listes de tâches et les roadmaps de ton projet ; en brut c'est éparpillé et tu perds la vision d'ensemble ; Roadmapped **ajoute l'interface** (un vrai dashboard live façon Monday/Jira/Linear) qui te donne la **vue d'ensemble** — backlog, ce qui est fait, ce qui est planifié — le tout restant des fichiers dans ton repo que tu review.

Copy retenue (fournie par Rémi, corrigée orthographe/grammaire ; `heil`→`hail`, filenames en minuscules monospace) :
- **Titre / hook** : `The roadmap.md is dead. All hail the task.yaml.`
- **Sous-titre** : `Transform your repo into your task management tool. Visualize your backlog, what's finished, and what's planned up ahead, and lead your development to success with a true big-picture view, powered by a Monday/Jira/Linear-like live project management dashboard.`

Le wordmark géant « Roadmapped » reste au-dessus. Répercuter le positionnement (dashboard live de gestion de projet pour ton repo, vue d'ensemble sur les plans de l'IA) sur la meta description et l'intro de How it works.
Écarté : centrer le hero sur « des fichiers YAML » (mécanisme, pas valeur) ; garder la tagline v2 (« we just added the interface » — juste, mais ne nomme pas la vue d'ensemble sur les plans IA).

### D2 — Hero : poids du wordmark
Le SVG « Roadmapped » reste pleine largeur mais passe de **bold → medium/semibold** (font-weight ~500-600) pour alléger. Idem tagline si elle est trop grasse.
Écarté : réduire la taille (Rémi veut garder gros).

### D3 — Première partie full-bleed AVEC grilles
Hero **et** démo passent en pleine largeur (bord à bord, plus de cap `--container-max` sur ces 2 sections). Les rails/colonnes façon Vercel restent **visibles** : au lieu de 2 filets aux bords du conteneur, une **grille de colonnes régulière** sur toute la largeur (filets verticaux répétés), pour que le full-bleed garde la structure. Le reste des sections garde le conteneur centré.
Écarté : full-bleed sans grille (perd la DA) ; tout le site en full-bleed (illisible pour le texte).

### D4 — Fond : trame de points (halftone) au lieu du pixel-art
Remplacer la tuile pixel-art par une **trame de points fins réguliers** (dot-grid / halftone), à la façon de la réf fournie (dégradé orange + fins points) : petits points à faible opacité, espacement régulier, éventuellement légère variation d'opacité en dégradé. Rendu CSS (`radial-gradient` répété) — léger, net, sans canvas. Compatible clair/sombre, ne gêne ni la lecture ni l'oiseau (reste en `z-index:-1`).
Écarté : garder le pixel-art (rejeté par Rémi) ; pattern lourd en image.

### D5 — Touches de bleu (avec légèreté)
On assouplit la doctrine v2 (« bleu = état/interaction seulement ») : on s'autorise le bleu comme **accent de DA** en plus des états — mais avec parcimonie (halos/traits/points de la trame teintés bleu, filets, petits détails). Orange reste réservé à l'oiseau. Objectif : plus vivant, pas chargé.
Écarté : bleu partout (trop) ; statu quo (Rémi veut plus).

### D6 — « How it works » : section pinned + scroll stepper
La section se **fige (sticky)** le temps que les **4 étapes défilent une par une au scroll** (progression pilotée par le scroll via IntersectionObserver/scroll-progress). Visuel/anim d'étape d'un côté, texte de l'étape de l'autre. Les 4 étapes = le flux réel (agent transforme l'intention/roadmap.md en tickets tracés → lit/écrit via CLI+skill → tu review le diff → dashboard live). **Fallback reduced-motion / sans JS** : les 4 étapes empilées, lisibles, sans pin.
Écarté : carrousel autoplay (moins immersif) ; garder les 4 étapes en texte statique (le retour de Rémi).

### D7 — Mettre à jour la démo embarquée (dernière version de l'app)
Régénérer la démo : `npm run build:demo` (repo app) → `dist-demo/` → recopier dans `site/demo/`. La démo embarque alors la dernière app (dont le dark mode #269).
Écarté : patcher la vieille build à la main (fragile).

### D8 — La démo suit le thème du site
Quand on bascule le site en sombre, l'iframe démo doit basculer aussi. Mécanique : l'iframe est même origine (`/demo/`). Le site passe le thème à l'iframe (query `?theme=dark|light` au chargement **et** message/`data-theme` au toggle) ; `index.demo.html` lit ce thème (script anti-flash, miroir de celui de l'app) et pose `data-theme` sur son `<html>`. Nécessite : (a) build à jour (D7), (b) petit script thème dans `index.demo.html` (repo app), (c) câblage côté site (passer + mettre à jour le thème de l'iframe au switch).
Écarté : forcer la démo en light (Rémi veut la synchro) ; deux domaines / postMessage cross-origin (inutile, même origine).

## Scope / hors-scope

**Dans le scope** : `site/index.html`, `site/style.css`, `site/roadmapped-bird.js` (si besoin pour le pin), `site/demo/` (rebuild), et côté app `index.demo.html` (script thème démo) + régénération `dist-demo`. Branche `site-v3`.
**Hors scope** : le dark mode de l'app dashboard (#269, fait par Rémi) et la dette #270 ; le contenu produit lui-même (features) ; SEO au-delà des metas déjà en place.
